# Roadmap

## Current milestone: 0.1 — Reliable foundation

Goal: make the checkpoint prototype safe to extend and straightforward to test.

- [x] Create the portrait Godot prototype
- [x] Publish the initial GitHub repository
- [x] Define architecture and narrative-data boundaries
- [x] Add persistent Codex project guidance
- [x] Add automated story validation
- [x] Add versioned saves and visible load errors
- [ ] Run the project in Godot on the owner's Windows laptop
- [ ] Confirm choice flow, restart, and persistence manually
- [ ] Create the `v0.1.0` GitHub release after acceptance

## 0.2 — Polished story reader

- Responsive panel and dialogue layout
- Tap-to-continue text progression
- Accessible choice presentation
- Scene transitions
- Save/load and settings screens
- Android back-button policy

## 0.3 — Chapter One vertical slice

- Final 20–30 minute chapter script
- Character and faction records
- Separate environment-first monochrome panels
- Sound, music, and atmosphere
- Multiple meaningful outcomes with delayed consequences

## 0.4 — Android test build

- Physical-device layout and performance testing
- Export presets and signing process
- Internal APK/AAB release
- Google Play closed-testing preparation

## Deferred

Combat, monetization, online accounts, cloud saves, and additional protagonists remain out of scope until the narrative vertical slice is enjoyable.
